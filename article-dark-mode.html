Dark Mode Design **Principles**

Creating depth with shadows and lighting without using pure black backgrounds for a truly comfortable user experience.

Oct 30, 2025 6 min read UI/UX

Beyond Pure Black: Why Dark Grey Matters

A common misconception in dark mode design is to use pure black (`#000000`) for backgrounds. While it saves battery on OLED screens, it can also create a stark contrast with lighter elements, causing eye strain. Instead, using a **dark grey or deep indigo** provides a softer visual and allows for better depth perception.

Layering for Depth

Just like in your glassmorphism design, layering is key. Different shades of dark grey can indicate elevation and hierarchy. Lighter dark shades appear to float higher, while darker ones recede.

:root {
    --dark-surface-0: #121212; /* Base background */
    --dark-surface-1: #1E1E1E; /* Raised cards */
    --dark-surface-2: #2C2C2C; /* Dialogs, modals */
}

/* Example of a raised card in dark mode */
.glass-card {
    background-color: var(--dark-surface-1);
    box-shadow: 0px 4px 10px rgba(0, 0, 0, 0.4);
}

Color Theory in Dark Interfaces

When adapting your brand colors to dark mode, avoid oversaturated hues. These can appear vibrant on light backgrounds but can become jarring on dark ones. Opt for **desaturated versions** of your primary colors or lighter, more muted tones.

Accessibility Considerations

Always ensure sufficient contrast between text and background. Tools like WebAIM Contrast Checker can help you meet accessibility standards (WCAG 2.1). Prioritize readability, especially for long-form content.
